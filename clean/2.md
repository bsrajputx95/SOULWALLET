## Observations

The account settings screen currently contains active session management UI (lines 475-533 in `file:app/account.tsx`) that displays device information, IP addresses, and allows users to revoke sessions. The session data is fetched via `trpc.auth.getSessions` and `trpc.auth.revokeSession` mutations. The `file:src/server/routers/auth.ts` router contains session-related endpoints (`getSessions`, `revokeSession`, `getSessionActivity`, `getLoginAttempts`) that need removal. The `file:components/account/SecurityModal.tsx` component exists but is not currently used in account.tsx, suggesting 2FA UI was already removed in a previous phase. The account.ts router contains device management endpoints (lines 689-793) which are separate from sessions and should be preserved.

## Approach

Remove all session management UI and backend endpoints while preserving essential account features (profile info, password reset button, invite friends, delete account). The active sessions section will be completely removed from the UI along with its associated TRPC queries. Backend session endpoints will be removed from the auth router, but the core session functionality (login/logout) will remain intact. The SecurityModal component will be deleted as it's unused. Device management endpoints in account.ts will be preserved as they serve a different purpose. This cleanup will simplify the account settings while maintaining all explicitly required functionality.
